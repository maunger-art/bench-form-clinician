# Benchmark PS Blog Engine

A fully automated blog content engine. Every Monday at 08:00 UTC, a new article is 
generated by Claude, added to the site, and deployed to GitHub Pages — with LinkedIn, 
Twitter/X, and newsletter snippets saved alongside it.

---

## Architecture

```
benchmark-blog/
├── blog_build.py          # Static site builder
├── generate_post.py       # Claude-powered article generator
├── social_snippets.py     # Social asset generator
├── posts_manifest.json    # All published posts (source of truth)
├── queue.json             # Upcoming article topics (ordered)
├── output/                # Built site — deployed to GitHub Pages
│   ├── index.html
│   ├── sitemap.xml
│   ├── feed.xml
│   ├── robots.txt
│   ├── posts/{slug}/index.html
│   └── social/{slug}/linkedin.txt, twitter.txt, insight.txt
├── requirements.txt
└── .github/workflows/
    ├── build.yml          # Deploy on push to main
    └── weekly_post.yml    # Monday 08:00 UTC cron job
```

## Quick Start

### 1. Clone and install

```bash
git clone https://github.com/YOUR_ORG/benchmark-blog.git
cd benchmark-blog
pip install -r requirements.txt
```

### 2. Configure brand variables

Open `blog_build.py` and update the configuration block at the top:

```python
SITE_NAME = "Benchmark PS"
SITE_URL = "https://benchmarkps.com/blog"   # Your actual domain
BRAND_PRIMARY_COLOR = "#1A3C5E"
GA4_ID = "G-XXXXXXXXXX"                     # Your GA4 measurement ID
GSC_VERIFICATION = "YOUR_TOKEN_HERE"        # From Google Search Console
```

### 3. Test the build locally

```bash
python blog_build.py
# Opens output/index.html in a browser to verify
open output/index.html
```

### 4. Generate your first post manually

```bash
export ANTHROPIC_API_KEY=sk-ant-...
python generate_post.py
```

This will:
- Take the first item from `queue.json`
- Call Claude to generate the article
- Append it to `posts_manifest.json`
- Rebuild the site in `output/`

Then generate social snippets:

```bash
python social_snippets.py
```

Social assets saved to `output/social/{slug}/`.

## GitHub Setup

### Step 1: Create a GitHub repository

Push this project to a new GitHub repository under your organisation.

### Step 2: Add the Anthropic API key as a secret

1. Go to your repo → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `ANTHROPIC_API_KEY`
4. Value: your `sk-ant-...` key

### Step 3: Enable GitHub Pages

1. Go to **Settings** → **Pages**
2. Set Source to **GitHub Actions**
3. Save

The first deployment triggers automatically on push to `main`.

### Step 4: Verify the cron job

Go to **Actions** → **Generate Weekly Post** — you'll see it scheduled for 
Mondays at 08:00 UTC. You can trigger it manually from the UI at any time 
using **Run workflow**.

## Google Search Console Setup

### Submit your sitemap

1. Go to [search.google.com/search-console](https://search.google.com/search-console)
2. Add property: `https://benchmarkps.com/blog`
3. Verify ownership using the HTML meta tag method:
   - Copy the verification token from GSC
   - Paste it into `GSC_VERIFICATION` in `blog_build.py`
   - Push to trigger a rebuild
4. Once verified, go to **Sitemaps** → Submit `https://benchmarkps.com/blog/sitemap.xml`

### Request indexing for new posts

After each new post is published, go to **URL Inspection** in GSC and 
request indexing for the new URL. GSC will typically crawl within 24-48 hours.

## Adding Articles to the Queue

Edit `queue.json` and add topic strings in the order you want them published:

```json
[
  "Your next article topic here",
  "The one after that",
  "..."
]
```

Push to `main`. The cron job picks them up automatically.

## Manual Controls

**Rebuild site without generating a new post:**
```bash
python blog_build.py
```

**Generate social snippets for a specific post:**
```bash
python social_snippets.py --slug your-post-slug
```

**Skip the next queue item** (edit queue.json and remove the first entry manually).

## Troubleshooting

**Build fails with `FileNotFoundError`**: Ensure `posts_manifest.json` and `queue.json` 
exist at the project root. Both are included in this repo.

**JSON parse error in `generate_post.py`**: Claude occasionally wraps output in markdown 
fences. The script strips these automatically, but if it fails, check the raw API response 
by adding `print(raw)` before the `json.loads()` call.

**GitHub Pages not updating**: Check the **Actions** tab for build errors. The most common 
cause is a missing or incorrect `ANTHROPIC_API_KEY` secret.

**GSC not indexing posts**: Submit the sitemap URL directly in GSC and use URL Inspection 
to request crawling for individual posts. New domains can take 2-4 weeks for initial indexing.
